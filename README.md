## Мобильное приложение с расписанием Кубанского Государственного Технологического Университета
Приложение получает данные о расписании напрямую из официальной базы данных для сторонних приложений университета и делает его доступным оффлайн!
- 📱 **Всё под рукой**: расписание прямо на мобильном устройстве, без необходимости поиска на сайте;
- 🗓️ **На любой случай**: удобный просмотр расписания занятий на текущий день, семестр и любую выбранную дату;
- 🎓 **Незаменимо во время сессии**: позволяет просматривать расписание любых преподавателей;
- 🇷🇺 **Актуально в современных реалиях**: позволяет сохранять доступ к расписанию всегда, даже когда официальный сайт ВУЗа недоступен из-за ограничений!

## Лицензия [Gplv3](./LICENSE)

## ✨ Основные возможности
- **Показ любого расписания**: Просмотр пар на текущий день и полный учебный семестр.
- **Экзаменационные сессии**: Отдельный блок для отслеживания дат и времени экзаменов.
- **Многопрофильность**: Поддержка как очной, так и заочной форм обучения, наличие расписания для всех структурных подразделений университета.
- **Offline-режим**: Полная работоспособность без интернета благодаря локальному кэшированию данных.
- **Подробная информация о парах**: место и время проведения, тип занятия, имя преподавателя и наименование предмета;

## 🛠️ Стек технологий
В проекте используется современный стек и подходы, обеспечивающие масштабируемость и тестируемость:
- **Flutter** - кроссплатформенный фреймворк с открытым исходным кодом, позволяет использовать единую кодовую базу на всех платформах;
- Локальная база данных реализована с использованием **Drift** - популярной библиотеки, поддерживающей разработку для всех распространённых платформ и реактивные запросы;
- Управление состояниями и разделение UI и логики реализовано с использованием библиотеки **Bloc**;
- Навигация реализована с использованием библиотеки с декларативным роутингом **go_router**;
- Сетевое взаимодействие реализовано при помощи библиотеки **Dio**, мощного пакета для разработки взаимодействия по протоколу HTTP;
- Автоматическая сериализация данных с кодогенерацией выполнена с применением библиотеки **json_serializable**;
- Инъекция зависимостей реализована при использовании библиотеки управления зависимостями **get_it**;

## 🏗️ Архитектура
Приложение спроектировано согласно принципам **Clean Architecture** (Чистая архитектура): код разделён на слои данных, бизнес-логики и отображения, что улучшает тестируемость, упрощает замену модулей и библиотек и уменьшает запутанность кода.

## 📂 Структура проекта
В проекте реализовано структурное **разделение по функциональным модулям**, что обеспечивает высокую масштабируемость, облегчает поиск и удаление отдельных блоков, упрощает совместную работу над проектом.

## 📐 Используемые паттерны проектирования
- **BLoC** - реализует разделение UI и бизнес-логики;
- **Инъекция зависимостей** - упрощает управление зависимостями в проекте, позволяя легко заменять функциональные модули при тестировании и смене технологий;
- **Singleton** - оптимизирует использование памяти, позволяя использовать один экземпляр объектов в разных частях приложения;
- **DTO** и **Mapper** - разделяет сущности бизнес-логики и классы-модели сторонних источников данных;
- **Репозиторий** - абстрагирует разные источники данных, позволяя использовать единый интерфейс для их получения;
- **Наблюдатель** - реактивное обновление интерфейса при изменении состояния;
